# Advantages of System design :
1.In good system design , it is easy to see how all parts of the system connect , interact and how they are perform together.
	eg:- chat-software
		module-1(Login )----> module-2(Chat)-----> module3(logout) 

2.It help teams to plan resource ,time and cost more efficient and reduces the failure of development.

3.An efficient system to maintain and update ,which save time and effort in the long run application.

4.it allows the system to grow smoothly .you can add new feature or handle more user with any major changes.

5. Good design can enhance the system performance ,making it faster and more efficient.


# Disadvantage of System Design :

1.The process of system design can be time-consuming ,especially for large and complex system .

2.System design can be expensive ,especially if it involve the significant researcher , prototyping , and testing.

3.System design requires a significant amount of technical expertise , including knowledge of hardware ,software ,database and data structure and algorithm . 

4. Once a system design is completed , it can be difficult to make changes to the design , especially if the design has already been implemented and deployed .

# What is HLD (High-level design)?
--> A bird's-eye view of the system , showing the major modules , data flow and component interaction without implementation details .

# Component of HLD :

	Component 					Description
--------------------------------------------------------------------------------------
1. System Architecture  -------------------------------> Overall System Blueprint

2. Module/Components	-------------------------------> Separated function (Eg: login, logout, sign -in ,payment etc .)

3.DFA (Data flow diagram )-----------------------------> Show how data moves through-out the system.

4.Interface Design ------------------------------------> Define how component interact (API , DB , UI etc. )

--> use diagram ,flowchart ,block layout.

# Purpose of HLD :

1.Help Visualizing major system flow .

2.Technical translation of functional requirements.

3. Enable collaboration between teams .

4. Support decision making for infrastructure and design . 


# Characteristic of HLD :-

1.Adaptable to changes :-
 
-> A good system design should support Change Control.

-> This means the architecture is flexible enough to adopt new requirement without redesigning everything.

-> Follow agile principle and knowledge of Version Control .


2.Visual-heavy :-

-> The Design include diagram and charts like:-
	-> DFD (Data flow diagram)
	-> UML (Unified Modeling language)
	-> ERD (Entity Relationship Diagram)
	-> Architecture Diagram .

->Helps developer and stakeholders /clients/user easily understand the system flow.

3. Maps FR(Functional requirement )to Modules :-

-> FR-> Functional Requirements -> What the system must do .

-> These are mapped to respective modules or component.
	eg : - FR: "User requirement" -> Mapped to " Authentication module "


4. Combination of HW + SW Layout .

-> HW - Hardware , SW- Software 

-> HLD Should include both 
	-> HW ( Server specification , cloud deployment , Storage etc.   )
	-> SW ( Microservices , APIs .backend/Frontend structure ).

# What is Low level Design ?

-> Detailed blueprint of each and every module , showing inter logic ,class Design and algorithm .

# include inside the LLD

-> UML (Unified modeling Language ).
-> Classes and sequence Diagram .
-> Pseudocode & Database Schema .

( LLD -------to--------HLD )


#Creating LLD from HLD

	Tool				Use 
----------------------------------------------------------
      	ULM 	--------------------->	class/Sequence Diagram etc.

 	OOPS	--------------------->	Abstraction , Encapsulation , Inheritance etc.

Example :- ATM Machine 
	1.Check balance - LLD
	2.debit balance -LLD
	3.Withdraw Case - HLD
	4.Print receipt.-LLD

# Difference between HLD and LLD

Criteria	|	      HLD			 	   |	LLD
----------------|--------------------------------------------------|--------
Based on 	|	SRS(Software requirement specific )	   |	HLD
		|
Audience 	|	Architects , PMs				Developer and tester
		|
Focus 		|	System Modules					internal logic of each
		|
Approach	|	Top-down					Bottom-up
		|
Tools		|	DFD , System diagram 				UML , class Diagram.
